# README screenshots — capture guide

Add PNG files here (recommended: **1440×900** desktop, **390×844** mobile).  
Use **light mode** unless noted. Hide real customer emails, phone numbers, and order IDs where possible.

| File | What to capture | Tips |
|------|-----------------|------|
| `01-hero-home.png` | Homepage — hero image, nav, announcement bar | Desktop, full browser width, scroll so hero + “Shop by Category” heading visible |
| `02-category-shop.png` | Any category page with products in a grid | e.g. `/category/wearables` — show filters/sort if visible |
| `03-product-detail.png` | Single product page | Image gallery, price, Add to Cart, Bengali/English labels |
| `04-checkout.png` | Checkout or cart drawer | Show COD / payment options UI (blur personal address if filled) |
| `05-mobile-home.png` | Homepage on phone | Chrome DevTools → iPhone 14 Pro (390px) or real phone screenshot |
| `06-admin-dashboard.png` | `/admin` dashboard | Revenue cards, order status, category chart |
| `07-admin-products.png` | Admin → Products list or edit form | Shows CRUD + image upload |
| `08-admin-orders.png` | Admin → Orders table | Status badges, order list |
| `09-dark-mode.png` | *(optional)* Storefront in dark mode | Same page as `01` but toggled dark |
| `10-track-order.png` | *(optional)* Track order page | Clean public utility page |

**How to screenshot (Windows):** `Win + Shift + S` → save into this folder with the exact filenames above.

After adding images, commit and push — they will appear in the GitHub README automatically.

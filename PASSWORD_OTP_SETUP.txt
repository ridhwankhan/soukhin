SOUKHIN — PASSWORD OTP SETUP
============================

You already ran the SQL (migration 016). Good.

STEP 1 — Deploy the edge function (pick ONE method)
-------------------------------------------------

METHOD A — npm (fastest if you have a token)

  1. Open: https://supabase.com/dashboard/account/tokens
  2. Generate new token → copy it
  3. Open PowerShell in the Soukhin folder:

     cd e:\Projects\Soukhin
     $env:SUPABASE_ACCESS_TOKEN = "sbp_paste_token_here"
     npm run deploy-password-otp

METHOD B — Supabase Dashboard (no CLI)

  1. Open: https://supabase.com/dashboard/project/yxctdtihkmslpidscfph/functions
  2. Deploy new function → Via Editor
  3. Function name: password-otp
  4. Open file: supabase\functions\password-otp\index.ts
  5. Copy ALL → paste in editor → Deploy


STEP 2 — Email secrets (optional but recommended)
-------------------------------------------------

  1. Open: https://supabase.com/dashboard/project/yxctdtihkmslpidscfph/settings/functions
  2. Under "Secrets", add (if not already there):

     RESEND_API_KEY          = your key from https://resend.com
     NOTIFICATION_FROM_EMAIL = Soukhin <onboarding@resend.dev>

  Without RESEND_API_KEY: PIN still works — it shows on screen after you request it.


STEP 3 — Test locally
---------------------

  cd e:\Projects\Soukhin
  npm run dev

  Forgot password:
    http://localhost:5173/auth → Sign In → Forgot password?

  Change password (signed in):
    http://localhost:5173/account → Change password


STEP 4 — Reset your admin password now
--------------------------------------

  1. Sign out (or use Incognito)
  2. /auth → Forgot password?
  3. Enter your admin email
  4. Check email for 6-digit PIN (or read PIN on screen if no Resend)
  5. Enter PIN + new password
  6. Sign in with new password


STEP 5 — Go live (when ready)
-----------------------------

  Push code to GitHub → Vercel redeploys automatically.
  The edge function is already on Supabase — no extra Vercel step for OTP.

  You do NOT need new Vercel env vars for password OTP.


TROUBLESHOOTING
---------------

  "Failed to send" / network error
    → Edge function not deployed. Do Step 1.

  "Invalid or expired PIN"
    → PIN expires in 10 minutes. Click Resend PIN.

  No email received
    → Check spam. Or use on-screen PIN (no Resend key).
    → Verify RESEND_API_KEY in Supabase Edge Function secrets.

  supabase not recognized
    → Use: npx supabase ...   or   npm run deploy-password-otp
